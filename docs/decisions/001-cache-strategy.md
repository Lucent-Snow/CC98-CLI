# ADR-001: 缓存策略

## 状态

已接受

## 背景

CC98 API 有请求频率限制，且 TUI 浏览场景需要快速响应。需要一个缓存层来：
1. 减少 API 请求
2. 提升 TUI 响应速度
3. 支持离线查看已缓存内容

## 决策

采用两级缓存架构：

### 内存缓存（LRU）
- 最大条目数: 500
- 淘汰策略: LRU（最近最少使用）
- 用于热数据快速访问

### 文件缓存
- 存储位置: `~/.cc98-cli/cache/`
- 文件名: SHA-256(缓存键)
- 格式: JSON，包含 `createdAt`、`expiresAt`、`value`
- 自动清理: 启动时后台清理过期条目

### TTL 配置
- 版面主题列表: 30s（高频变动）
- 版面信息: 24h（低频变动）
- 用户信息: 5min
- 其他: 默认 60s

## 后果

### 正面
- TUI 浏览体验流畅
- 减少 API 负载
- 支持快速刷新（force=true 跳过缓存）

### 负面
- 缓存过期时可能看到旧数据
- 文件缓存占用磁盘空间（已限制 7 天自动清理）
- TTL 硬编码，无法用户配置

- `src/storage/cache-store.ts` - 缓存实现
- `src/tui/cached-client.ts` - TTL 配置
