# TUI 设计分析

> 更新时间：2026-05-27

## 一、整体架构

### 1.1 技术栈

- **渲染**: ANSI 转义码，纯文本终端渲染
- **状态管理**: 单一状态对象 `TuiState`
- **事件处理**: 终端键盘输入监听
- **数据层**: `CachedCc98Client`（带缓存的 API 客户端）

### 1.2 文件结构

```
src/tui/
├── app.ts              # 入口：终端生命周期和依赖组装
├── controller.ts       # 按键处理、导航、加载和写操作副作用
├── renderer.ts         # 三栏布局、模态框和组件组合
├── navigation.ts       # 左栏导航和设置项配置
├── helpers.ts          # API 响应归一化和列表项转换
├── topic-reader.ts     # 帖子楼层、UBB 行和右栏索引
├── components/         # Header/Overview/Sidebar/Content/StatusBar
├── state/              # TuiState 类型、初始状态、默认状态文案
├── terminal.ts         # 终端 I/O 抽象
├── ansi.ts             # ANSI 转义码工具
├── cached-client.ts    # 带缓存的 API 客户端
└── ubb-renderer.ts     # UBB 代码渲染器
```

### 1.3 核心流程

```
用户按键 → TuiController → 状态更新 → render()
              ↓
          API 调用（可选）
              ↓
          状态更新 → renderer/components → 终端输出
```

## 二、状态管理

### 2.1 状态结构

```typescript
interface TuiState {
  // 视图状态
  mode: "list" | "topic" | "settings" | "user-detail";
  focus: "nav" | "content";
  navIndex: number;          // 当前导航项索引
  itemIndex: number;         // 当前内容项索引
  scroll: number;            // 滚动位置

  // 数据状态
  loading: boolean;
  loadingMore: boolean;
  error?: string;
  account?: string;

  // 视图数据
  viewTitle: string;
  items: ContentItem[];      // 当前列表项
  stats: ContentItem[];      // 右栏统计
  overview: ContentItem[];   // 顶部概览

  // 导航状态
  parentList?: ListSnapshot; // 父列表快照（用于返回）
  currentBoard?: BoardListState;
  currentChat?: ChatListState;
  topic?: TopicReaderState;

  // 模态框状态
  modal: "menu" | "help" | "search" | "user" | "info" | null;
  menuIndex: number;
  menuItems: MenuItem[];

  // 新增状态
  tabId: TabId;
  tabs: { id: TabId; label: string }[];
  searchMode: "topics" | "users";
  searchQuery: string;
  searchResults: ContentItem[];
  userDetail?: UserDetailState;
  noticeType: "system" | "at" | "reply";
  inputMode: boolean;
  inputPrompt: string;
  inputValue: string;
  inputCallback?: (value: string) => void;
  infoTitle?: string;
  infoLines: string[];
}
```

### 2.2 状态设计优点

- ✅ **单一状态源**: 所有状态集中管理，易于理解和调试
- ✅ **集中副作用**: `TuiController` 持有 API、AbortController、render 回调，避免组件直接请求
- ✅ **父列表快照**: 支持返回上一级导航

### 2.3 状态设计问题

- ⚠️ **状态仍然偏扁平**: 为了简单可调试，搜索、输入、用户详情、通知状态仍在同一个 `TuiState`
- ⚠️ **模式仍需收敛**: `mode`, `focus`, `modal`, `inputMode` 共同决定交互上下文
- ⚠️ **写入操作缺少确认层**: 收藏、点赞、关注等操作已接入，但还没有统一的确认/撤销体验

## 三、视图结构

### 3.1 导航项（10 个）

| ID | 标签 | 提示 | 说明 |
|----|------|------|------|
| hot | 十大 | 热门话题 | 首页热帖 |
| favorite | 收藏 | 版面帖子 | 收藏的版面和帖子 |
| new | 最新 | 新帖流 | 最新发布的帖子 |
| boards | 版面 | 所有分区 | 版面列表 |
| following | 关注 | 用户动态 | 关注用户的帖子 |
| messages | 消息 | 未读与私信 | 私信列表 |
| notices | 通知 | 系统与回复 | 通知列表 |
| me | 我的 | 当前账号 | 个人信息和功能 |
| more | 更多 | 只读内容 | 其他只读功能 |
| settings | 设置 | 账号与配置 | 应用设置 |

### 3.2 视图内容

#### 十大 (hot)
- **数据**: 首页配置 + 未读消息
- **内容**: 热帖列表
- **概览**: 今日主题、回复、在线、用户数

#### 收藏 (favorite)
- **数据**: 用户信息 + 版面列表 + 收藏主题
- **内容**: 收藏主题入口 + 收藏分组入口 + 收藏主题 + 版面帖子
- **统计**: 收藏版面数、收藏主题数、版面主题数

#### 最新 (new)
- **数据**: 最新帖子
- **内容**: 新帖列表

#### 版面 (boards)
- **数据**: 所有版面
- **内容**: 版面列表
- **进入后**: 版面帖子列表

#### 关注 (following)
- **数据**: 关注动态
- **内容**: 关注用户的帖子

#### 消息 (messages)
- **数据**: 未读消息 + 最近联系人
- **内容**: 私信列表
- **进入后**: 私信对话

#### 通知 (notices)
- **数据**: 未读消息数
- **内容**: 系统通知、@通知、回复通知入口
- **进入后**: 通知列表

#### 我的 (me)
- **数据**: 用户信息 + 缓存统计
- **内容**: 用户信息 + 功能列表
- **功能**: 签到、最近主题、浏览历史、粉丝、关注

#### 更多 (more)
- **数据**: 无
- **内容**: 只读功能入口列表
- **功能**: 随机主题、收藏主题、收藏更新、收藏分组、粉丝、关注、全站统计、评分原因

#### 设置 (settings)
- **数据**: 缓存统计
- **内容**: 设置项列表
- **功能**: 切换账号、检查更新、缓存管理、帮助、退出登录

## 四、布局设计

### 4.1 三栏布局

```
┌──────────┬────────────────────────────────┬──────────────────┐
│ 左栏导航 │ 中栏内容                        │ 右栏信息         │
│ (14-18字符) │ (自适应)                        │ (34-42字符)      │
└──────────┴────────────────────────────────┴──────────────────┘
```

### 4.2 响应式设计

- **宽度 < 78**: 隐藏右栏
- **宽度 < 90**: 左栏 14 字符
- **宽度 >= 90**: 左栏 18 字符
- **高度 < 24**: 概览区 1 行
- **高度 >= 24**: 概览区 2 行

### 4.3 布局优点

- ✅ **空间利用**: 三栏布局充分利用横向空间
- ✅ **响应式**: 根据终端大小调整
- ✅ **信息层次**: 导航、内容、详情分离

### 4.4 布局问题

- ❌ **右栏利用率低**: 很多视图右栏只有简单统计
- ❌ **概览区固定**: 始终占用 1-2 行空间
- ⚠️ **状态栏信息密**: 已压缩为按上下文展示，但窄终端仍可能拥挤

## 五、交互设计

### 5.1 模式系统

| 模式 | 说明 | 快捷键 |
|------|------|--------|
| list | 列表浏览 | j/k 移动, l 进入, h 返回 |
| topic | 帖子阅读 | j/k 滚动, n 下页, 【/】楼层 |
| settings | 设置浏览 | j/k 移动, l 执行 |
| user-detail | 用户详情 | f 关注, m 私信 |

### 5.2 模态框系统

| 模态框 | 触发 | 功能 |
|--------|------|------|
| help | `?` | 快捷键帮助 |
| menu | `o` | 上下文操作菜单 |
| search | `/` | 搜索帖子/用户 |
| user | `u` | 用户详情 |
| info | - | 信息展示 |

### 5.3 输入模式

- **触发**: 发私信时
- **显示**: 状态栏显示输入提示
- **操作**: 输入、删除、确认、取消

### 5.4 交互优点

- ✅ **Vim 风格**: j/k/h/l 导航
- ✅ **上下文菜单**: `o` 键打开操作菜单
- ✅ **模态框系统**: 搜索、用户详情等
- ✅ **退出和请求取消**: 切换视图、返回和退出会中止当前请求，避免过期渲染

### 5.5 交互问题

- ⚠️ **模式切换复杂**: 不同模式快捷键不同，需要继续打磨帮助文案
- ❌ **输入模式简陋**: 只支持单行输入
- ⚠️ **返回逻辑复杂**: h/Esc/方向键已集中到控制器，但仍需要更多交互测试

## 七、数据流设计

### 7.1 数据加载

```
loadView(client, viewId, force, signal)
  ↓
API 调用（带缓存）
  ↓
数据转换（asArray, asObject, topicItem, chatItem）
  ↓
返回 { title, items, stats, overview, status }
  ↓
更新 state
  ↓
重新渲染
```

### 7.2 缓存策略

| 数据类型 | TTL | 说明 |
|----------|-----|------|
| 首页配置 | 30s | 高频变动 |
| 版面列表 | 24h | 低频变动 |
| 版面帖子 | 30s | 中频变动 |
| 用户信息 | 5min | 低频变动 |
| 帖子详情 | 60s | 中频变动 |
| 私信历史 | 15s | 高频变动 |

### 7.3 数据流优点

- ✅ **缓存优先**: 减少 API 请求
- ✅ **并发请求**: `Promise.all` 并行加载
- ✅ **请求取消**: `AbortController` 支持取消

### 7.4 数据流问题

- ❌ **错误处理简单**: 只显示错误信息
- ❌ **无重试机制**: 失败后需手动刷新
- ❌ **无离线支持**: 无网络时无法使用

- ❌ **中文宽度**: 双宽度字符处理复杂
- ❌ **滚动实现**: 手动计算滚动位置

## 九、设计问题总结

### 9.1 架构层面

1. **状态管理混乱**: 状态过于扁平，缺乏组织
2. **模式系统复杂**: mode、modal、inputMode 混用
3. **组件化缺失**: 所有逻辑在一个文件

### 9.2 交互层面

1. **快捷键冲突**: 不同模式下相同键功能不同
2. **返回逻辑复杂**: h/Esc/Backspace 都可能返回
3. **输入模式简陋**: 只支持单行输入

### 9.3 视图层面

1. **导航项过多**: 10 个导航项，需要滚动
2. **右栏利用率低**: 很多视图右栏只有简单统计
3. **功能分散**: 某些功能在"我的"，某些在"更多"

### 9.4 数据层面

1. **错误处理简单**: 只显示错误信息
2. **无重试机制**: 失败后需手动刷新
3. **无离线支持**: 无网络时无法使用

## 十、改进建议

### 10.1 短期（1-2 天）

1. **整理导航项**: 合并"消息"和"通知"，减少到 8 个
2. **优化状态栏**: 只显示当前模式的快捷键
3. **改进右栏**: 为空时显示提示信息

### 10.2 中期（3-5 天）

1. **重构状态管理**: 按模式组织状态
2. **统一返回逻辑**: 只用 Esc 返回
3. **改进输入模式**: 支持多行输入

### 10.3 长期（1 周+）

1. **组件化重构**: 拆分独立组件
2. **性能优化**: 增量更新
3. **主题系统**: 支持自定义主题
